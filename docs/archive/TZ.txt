1.1. LOYIHA HAQIDA UMUMIY MA'LUMOT (PROJECT OVERVIEW)
TenderHelper — O‘zbekiston davlat xaridlari va korporativ xaridlar bozorida faoliyat yurituvchi, kichik va o‘rta biznes (SME) sub'ektlari uchun maxsus yaratilgan AI Tender Mentor (Sun'iy Intellektual Assistant) platformasidir.

Muammo (The Pain Point):
O‘zbekistonda davlat va korporativ tenderlar hajmi yiliga trillionlab so‘mni tashkil etsa-da, kichik tadbirkorlarning aksariyati byurokratiya, murakkab yuridik til, doimiy diskvalifikatsiya (hujjatdagi xatolar tufayli rad etilish) va auksionlardagi hissiy demping (zararga narx tushirib bankrot bo‘lish) sababli bu bozordan quruq qolmoqda. Mavjud bozor yechimlari (masalan, TenderZone, Tender.uz) faqat o‘tgan tarixiy statistikani ko‘rsatuvchi passiv filtr xolos.

Yechim (The Core Value):
TenderHelper bozordagi bo‘shliqni to‘ldirib, "Biz tender platformasi emasmiz, biz tender yutishga yordam beruvchi shaxsiy mentormiz" falsafasi asosida ishlaydi. Tizim foydalanuvchining raqamli identifikatori (profil ko‘rsatkichlari) bilan jonli lot shartlarini parallel tahlil qilib, tadbirkorga real yordam va strategik yo‘nalish beradi.

PostgreSQL (LIKE / __icontains qidiruvi): MVP bosqichida murakkab qidiruv tizimlarini (Elasticsearch va hk) o‘rnatib vaqt yo‘qotmaslik va algoritmik murakkablikni optimallashtirish uchun Django ORM ning ichki matnli qidiruv imkoniyatlaridan maksimal foydalaniladi.

3.3. PWA (PROGRESSIVE WEB APP) STRATEGIK XUSUSIYATLARI
Ilova brauzerda ham, telefonga oʻrnatilganda ham mobil-ilova (Native) kabi ishlashi uchun React jamoasi quyidagi optimal PWA qoidalarini kiritishi shart:

📌 4-QISM: BACKEND ARXITEKTURASI, API INTEGRATSIYASI VA ASINXRON JARAYONLAR (CELERY/REDIS)
Ushbu boʻlim Django REST Framework (DRF) arxitekturasini toʻgʻri qurish, davlat xaridlari portallaridan lotlarni uzluksiz yuklash va LangChain/RAG modullarini tizimga xavfsiz integratsiya qilish mexanizmlarini belgilab beradi.

4.1. DATA AGGREGATION & PIPELINE (SKREPING VA MA'LUMOTLAR OQIMI)
Tizimning jonli ishlashi va lotlarning doimiy yangilanib turishi Celery (Beat) va Redis orqali asinxron rejimda boshqariladi. MVP yuklamasini kamaytirish uchun ma'lumotlar oqimi quyidagi davriylikda ishlaydi:

1. Avtomatik Lot Skreperi (Periodic Tasks)
Ishlash vaqti: Har 30 daqiqada bir marta (Celery Beat crontab orqali).

Vazifasi: xarid.uzex.uz, dxarid.uzex.uz, exarid.uzex.uz va e-auksion.uz portallarining ochiq ma'lumotlar oqimi (RSS, JSON API yoki HTML parser) skanerlanadi.

Mantiq: Faqat oxirgi tekshiruvdan keyin qoʻshilgan, statusi active boʻlgan yangi lotlar bazaga yoziladi. Agar lot raqami TenderLot jadvalida allaqachon mavjud boʻlsa, uning muddati (deadline) va joriy narxi yangilanadi (upsert logikasi).

2. Hujjatlarni Avtomat Yuklash (Background Document Downloader)
Foydalanuvchi ixtiyoriy lotni tanlab, "Tahlilni boshlash" tugmasini bosganda, React frontend API-ga soʻrov yuboradi.

Django backend bu ogʻir jarayonni HTTP soʻrov ichida bajarmaydi (timeout boʻlmasligi uchun) va uni darhol Celery Worker navbatiga (Queue) tashlaydi.

Celery Worker lotga biriktirilgan texnik spetsifikatsiya va nizom fayllarini (.pdf, .docx) yuklab oladi, matnga oʻgiradi va TenderDocumentChunk jadvaliga boʻlaklab saqlaydi.

4.2. API ENDPOINTS MATRIX (BACKEND API SHINASI)
Frontend va backend oʻrtasidagi aloqa toʻliq JSON formatida, DRF TokenAuthentication (yoki JWT) orqali himoyalangan holda amalga oshiriladi.

🔐 1. Auth & Profile Endpoints
POST /api/v1/auth/oneid/ — OneID login tokenini qabul qilish va foydalanuvchini avtomat roʻyxatdan oʻtkazish/tizimga kiritish.

POST /api/v1/company/onboarding/ — 3 ta onboarding savollari javobini qabul qilish va profilni mahkamlash.

GET /api/v1/company/profile/ — Raqamli dosye va oʻtmishdagi tenderlar statistikasini qaytarish.

🧭 2. Tender & Search Endpoints
GET /api/v1/tenders/ — Barcha lotlar roʻyxati.

Qidiruv logikasi (Django Backend):

Python
queryset = TenderLot.objects.filter(status='active')
search_query = request.query_params.get('search', None)
if search_query:
    # Django ORM orqali LIKE / __icontains qidiruvi
    queryset = queryset.filter(Q(title__icontains=search_query) | Q(buyer_name__icontains=search_query))
Filtrlar: ?region=Toshkent&min_price=10000000&source=xarid_uzex formatidagi 25+ parametrlar shu endpointda filtrlanadi.

🤖 3. AI Analysis & Calculator Endpoints
POST /api/v1/analysis/start/ — Lot ID va foydalanuvchi yuklagan manfaatli hujjatlarni qabul qilib, asinxron AI tahlilini (AITenderAnalysis) boshlash.

GET /api/v1/analysis/<uuid:id>/status/ — React progress-bar uchun joriy holatni (processing_docs, checking_compliance, completed) qaytaruvchi endpoint.

POST /api/v1/analysis/<uuid:id>/calculate/ — Tadbirkor kiritgan xarajatlar asosida amaldagi 12% QQS va 0.15% birja komissiyasini hisoblab, Stop-Loss chizigʻini qaytarish.

GET /api/v1/analysis/<uuid:id>/export-proposal/ — Tayyorlangan tijorat taklifini Word/PDF koʻrinishida yuklab olish.

4.4. SECURITY & EXCEPTION HANDLING (XAVFSIZLIK VA XATOLIKLARNI BOSHQARISH)
1. API Rate Limiting (Hujum va Yuklamadan Himoya)
Anvondor foydalanuvchilar (Freemium): Kuniga maksimal 3 ta AI tahlil soʻrovi va daqiqasiga koʻpi bilan 15 ta qidiruv soʻrovi (AnonRateThrottle).

PRO va Enterprise: Cheksiz soʻrovlar, lekin tizimni skreper botlardan asrash uchun daqiqasiga maksimal 100 ta soʻrov cheklovi (UserRateThrottle).

2. AI Hallucination & Timeout Guard (Xatoliklar ssenariysi)
Agar tashqi AI API (OpenAI/Gemini) vaqtinchalik uzilib qolsa yoki 10 soniya ichida javob bermasa, tizim foydalanuvchiga xatolik koʻrsatmaydi. AITenderAnalysis statusi failed holatiga oʻtadi va Celery avtomatik ravishda soʻrovni yana 1 marta qayta bajarishga urinadi (retry mexanizmi).

AI qaytargan javob JSON formatida boʻlmasa, backend ichidagi SafeJSONParser uni tozalab, standart koʻrinishga keltiradi, bu esa frontend (React) arxitekturasining qulab tushishini (crash boʻlishini) 100% oldini oladi.
